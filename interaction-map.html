実行対象の決まり方

キーを押した場所より、残っている mark と tab 1 の cursor が強く影響します。

i install / delete plan

  1. selected があれば全件filter 外の hidden mark も含む。
  2. なければ tab 1 の現在 source 行focus が tab 0/2/3/4 でも同じ。
  3. provMarked を install 先に使用provider cursor 単体ではない。
  4. provDelete も plan に追加i でも delete が混ざる。
  5. localMarked は無視footer の marked 数とは一致しない。

p automatic PR

  1. localMarked があれば最優先現在 source 行や focus を上書きする。
  2. なければ現在 local-only 行destination を自動推定。
  3. なければ現在 source 行edited copy がある場合だけ PR plan。
  4. tab 2 focus なら provider cursor を優先copy がなければ別 provider へ fallback。
  5. selected source marks は無視source edit PR は batch されない。

d delete mark

  1. tab 2 だけ現在 provider を切替tracked install がある場合だけ。
  2. それ以外は source context 全体selected があれば全件。
  3. 未選択時の自動 mark は tab 1 だけtab 0/3/4 では cursor source を mark しない。
  4. 既存 install marks を退避もう一度 d で復元。
  5. 実行は enter または idelete 専用キーでは実行しない。

[x] の現在の意味

同じ記号が選択・状態・存在確認の3用途に使われています。

[x] skill-name

tab 0 / tab 1

原則として「選択済み」。ただし action に応じて [n]、[u]、[d] などへ変わります。

[x]✓ provider

tab 2 / source skill

「導入済みで予定 action なし」。provider が選択済みかどうかは判別できません。

[x]+ provider

tab 2 / local-only

「その provider directory にコピーが存在」。space を押しても切り替わりません。

モード中に上書きされるキー

通常画面の global key よりモード処理が先に評価されます。

mode 有効なキー 通常キーへの影響
loading / load error q esc ctrl+c i、p、移動などはすべて無視。
search enter keep、esc clear、ctrl+u clear、文字入力 i、p、d も検索文字として入力される。
confirm enter/y execute、esc/q/n cancel、j/k scroll tab 移動、mark、scope 切替などは無効。
destination picker j/k、enter、esc、文字入力 i、p などは入力欄では通常文字になる。

統一後の最小契約案

focus は navigation と space の対象だけを決め、実行キーの意味は固定します。
