Legacy visual reference

redesign 前の記号と組み合わせ。

このページは複雑化していた旧レンダラーの比較資料です。現在は角括弧を [ ] / [x] の選択だけに簡略化しました。 現在の最終仕様はこちら。

[u]↓guide_dpt_modelupdate

読み方: [action]state name optional-action-detail
01 / ACTION [u]

角括弧の中。Enterしたときの予定アクションです。

02 / CURRENT STATE ↓

角括弧の直後。現在の導入状態を1文字で集約します。

03 / TARGET skill-name

skillまたはprovider。右端には new / update / delete などが付きます。

State column

現在状態は8種類

通常行では角括弧の直後に表示。+ だけは local-only 専用、空白は「該当コピーなし」です。

✓

managed

許可ソースから導入済みで、ソースと同じ最新版。

↓

outdated

導入後にソース側が更新された。更新候補。

m

modified

導入済みコピーをローカルで編集した。PR提案可能。

~

partial

対象の一部だけ導入済み。集約表示専用。

!

foreign

別リポジトリ由来の同名コピー。ポリシー違反。

?

untracked

追跡メタデータのない同名コピー。手動配置。

+

local-only

インストール先にだけ存在し、ソースにはないskill。

 

blank / absent

該当する導入済み・foreign・untrackedコピーがない。

!

優先順位: 集約表示は m → ↓ → ✓ → ~ → ! → ? → 空白。 ただし同じ1コピーが「ソース更新あり＋ローカル編集あり」の両方なら、個別判定では ↓ が先に選ばれます。

[ ]↓ guide

skillは未選択。ただし選択中provider上のコピーは更新あり。

[n] guide

対象providerへ新規インストールする。

[n]~ guide

一部providerには導入済み。不足先へ新規導入する。

[u]↓ guide

outdatedコピーを更新する。

[u]m guide

ローカル編集済みコピーを再インストールで更新する。上書きに注意。

[D]✓ guide

現在scopeにある追跡済みコピーをすべて削除する。

[d]✓ guide

一部の導入先だけ削除し、別のコピーは残す。

[*]~ guide

providerごとにinstall/updateとdeleteが混在している。

[x]✓ guide

skillは選択済みだが最新版なので実作業なし。

[x]! guide

複数skillのaction集約で、この行自体には作業がなくforeignだけ検出。

i

Panel 1の状態列は全インストール先の総合監査ではありません。 現在install/delete対象になっているproviderだけを集約します。providerの選択を変えると、同じskillでも記号が変わります。

Panel 2 / providers

provider行の全組み合わせ

provider行では、状態が「現在選択中のskill集合」を集約します。右端の英語ラベルが実作業の内訳です。

action ＼ state 空白 ✓ ↓ m ~ ! ?
[ ]none [ ]  — — — — [ ]! [ ]?
[n]new [n]  — — — — [n]! [n]?
[u]update — [u]✓ [u]↓ [u]m [u]~ — —
[f]force [f]  [f]✓ [f]↓ [f]m [f]~ [f]! [f]?
[d]delete [d]  [d]✓ [d]↓ [d]m [d]~ [d]! [d]?

通常操作で表示 delete指定後に対象skillを切り替えた場合などに表示 — 現行ロジックでは生成しない
[n] codex new

未導入providerに新規インストール。

[n]! codex new

foreignの同名コピーを検出しているが、予定アクションはnew。

[u]~ codex new 1/2

2 skill中1件だけ不足。角括弧は u でも、実作業は右端のnew。

[u]~ codex new+upd 2/3

新規と更新が混在。対象は3件中2件。

[u]↓ codex update

outdatedコピーを通常更新。

[u]✓ codex reinstall

全体forceが有効。最新コピーも再インストール。

[f]✓ codex reinstall

このproviderだけforce指定。

[d]m codex delete

ローカル編集済みコピーを削除予定。内容消失に注意。

[x]↓ codex

outdatedだがprovider未選択なので、現在は操作なし。

[ ]? codex

追跡なし同名コピーはあるが、追跡済み導入も操作予定もない。

!

[u]~ ... new は矛盾ではありません。 角括弧の u は「このproviderに既存コピーがある状態でinstall actionがある」、右端の new は「今回実際に不足分へ行う作業」を示しています。

Special rows & auxiliary marks

local-only、tree、集計、補助記号

同じ文字でも、角括弧の内外や表示パネルによって意味が変わります。

1 skills / local-only
[ ]+
local-only skill。追加PRの一括対象には未選択。
[x]+
追加PRの一括対象として選択済み。
×N
現在scopeに存在する同名コピー数。
2 providers / local-only
[x]+
このproviderの配置先にlocal-onlyコピーが存在。
[ ] 
このproviderにはコピーが存在しない。
→
PR元として使われるコピー。
0 tree
[ ]
配下skillが未選択。
[x]
配下skillをすべて選択。
[~]
配下skillの一部だけ選択。
(N)
配下にあるsource skill数。
3 installed
u / p
user scope / project scope。
✓N
managedコピー数。
!N
foreignコピー数。
?N
untrackedコピー数。
u claude ✓5 !0 ?2 p codex+kimi ✓1 !0 ?0

codex+kimi の + はlocal-only記号ではなく、複数providerが同じ物理ディレクトリを共有しているという区切りです。

A / M / D

PR・diffのファイル一覧で Added / Modified / Deleted。

→ / ←

操作対象・PR元・実行コマンド / 現在scope。

✗ / ✓ / ⚠

削除または失敗 / 成功 / 衝突やoutdated警告。

-

previewの installed@scope で、そのproviderに該当コピーがない。

(+N @scope)

現在とは反対のscopeにあるlocal-onlyコピー数。

[UNAPPROVED SOURCE]

現在のsourceが許可リストに入っていないというヘッダー警告。

!

現在の既知の表示差: treeの仮想 (local-only) ノードは、spaceで内部の一括マークが変わっても角括弧が常に [ ] のままです。

現行の tools/gh-skill-tui/tui.go に基づくオフラインHTML。外部フォント・JavaScript・ネットワーク接続は不要です。
