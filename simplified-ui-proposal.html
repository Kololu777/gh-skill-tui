Implemented interaction model · 2026-07-10

選択は一種類。操作は plan で決める。

角括弧から action code をなくし、現在状態と実行内容を分離しました。source 外のコピーはすべて O outside に集約し、互換性チェックが一件でも失敗すれば選択全体を止めます。

[ ] / [x] = selection only i = Install / Update / Adopt p = one atomic PR d = managed Delete Enter = detail / execute plan

BLOCKED は理由単位で集約: 同じ診断文を skill ごとに繰り返さず、理由を一度だけ表示して、その下に対象 skill を列挙します。

覚えるルールは四つ

[ ] / [x]

中立選択

skill と agent のブラケットは対象を選ぶだけ。tree の [~] は部分選択だけです。

✓ ↓ m O

現在状態

managed current、outdated、edited、outside。導入範囲は記号ではなく N/M。

i / p / d

操作を解決

同じ選択を Install、PR、Delete のどれとして扱うかは押した operation key が決めます。

BLOCKED

部分実行なし

選択に blocker が一件でもあれば、Ready 項目も含めて enter は何も実行しません。

三つの操作 plan

Ready

  • INSTALL source にあり destination にない
  • UPDATE ↓ を non-interactive overwrite
  • ADOPT 同名 outside コピーを source 版へ明示的に置換

Blocking

  • O outside 行そのもの
  • m に overwrite override がない
  • agent が未選択

Ready

  • m の編集差分
  • O の source への追加
  • 両者を 1 branch / 1 commit / 1 PR に統合

Blocking

  • 未編集 source 行
  • 追加先の衝突・重複
  • 複数コピーの PR 元が曖昧

Ready

  • 追跡 metadata が source と一致
  • trusted destination root 配下
  • m は消える編集を warning 表示

Blocking

  • O は常に保護
  • 選択 agent に managed コピーがない
  • source 対応を検証できない
